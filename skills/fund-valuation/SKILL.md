# 基金估值系统技能

## 触发场景

当用户请求以下内容时，应使用此技能：
- 查询基金信息、净值、持仓
- 获取基金实时估值和涨跌幅
- 查询股票、ETF、指数行情
- 分析基金投资价值
- 批量估值多只基金
- **分析基金涨跌原因**
- **分析基金波动因子/风险来源**
- **解释金融术语/概念**
- **查询金融知识/最新资讯**
- **调研相关开源项目/论文**

## 能力说明

本技能提供以下功能：

| 能力 | 描述 | HTTP API |
|------|------|----------|
| 基金查询 | 获取基金信息、持仓、净值 | `GET /api/funds/{code}` |
| 实时估值 | 盘中实时估算基金净值 | `GET /api/valuation/{code}` |
| 持仓管理 | 添加/删除/查看持仓 | `GET/POST /api/funds` |
| 股票行情 | A 股实时价格 | `GET /api/market/stock/{code}` |
| ETF 行情 | 场内 ETF 实时价格 | `GET /api/market/etf/{code}` |
| 指数行情 | 国内外指数点位 | `GET /api/market/index/{code}` |
| 批量估值 | 同时估值多只基金 | `POST /api/valuation/batch` |

## AI 分析能力

除上述基础数据获取能力外，本技能还强调 AI 的分析推理能力，用于深度分析基金涨跌原因和波动因子。

### 涨跌原因分析

当用户要求分析基金涨跌原因时，AI 应按以下框架进行分析：

**1. 持仓结构分析**
- 获取基金前十大持仓（通过 `GET /api/funds/{code}/holdings`）
- 分析行业分布（消费、医疗、科技、金融等）
- 分析市值分布（大盘、中盘、小盘）
- 分析资产类型（A股、港股、美股、债券等）

**2. 贡献度分解**
计算每个持仓对基金净值变化的贡献：
```
贡献 = 持仓权重 × 涨跌幅
```
- 识别正贡献持仓（上涨贡献）
- 识别负贡献持仓（下跌拖累）
- 按行业汇总贡献

**3. 风格归因**
基于持仓特征判断基金风格：
- 价值风格 vs 成长风格
- 大盘蓝筹 vs 中小盘
- 周期行业 vs 防御行业

**4. 资金流向分析**
结合市场整体资金流向判断：
- 主力资金流入/流出方向
- 板块轮动规律
- 市场情绪判断

**输出示例：**
```
## 涨跌原因分析

### 今日涨跌幅：+1.23%

### 持仓贡献分解
| 持仓 | 权重 | 今日涨跌 | 贡献度 |
|------|------|----------|--------|
| 贵州茅台 | 8.5% | +2.1% | +0.18% |
| 宁德时代 | 6.2% | -1.5% | -0.09% |
| ... | ... | ... | ... |

### 行业贡献汇总
- 食品饮料：+0.25%（正贡献）
- 新能源：-0.15%（负贡献）

### 风格判断
该基金属于**大盘成长**风格，主要受益于今日消费板块反弹。

### 结论
今日上涨主要来自食品饮料行业的贡献，新能源板块的下跌部分抵消了涨幅。
```

### 波动因子分析

当用户要求分析基金波动因子时，AI 应识别以下风险来源：

**1. 系统性风险（Market Beta）**
- 市场整体涨跌对基金的影响
- 计算方式：基金涨跌幅 / 指数涨跌幅

**2. 行业风险（Industry Risk）**
- 各行业配置比例
- 行业轮动带来的波动

**3. 风格因子风险（Style Factors）**
| 因子 | 描述 | 影响方向 |
|------|------|----------|
| 市值因子 (Size) | 大盘股 vs 小盘股 | 小盘波动更大 |
| 价值因子 (Value) | 低估值 vs 高估值 | 价值股更稳健 |
| 动量因子 (Momentum) | 趋势延续性 | 追涨杀跌风险 |
| 波动率因子 (Volatility) | 波动大小 | 高波动基金回撤大 |
| 盈利因子 (Profitability) | 盈利质量 | 优质公司更稳定 |

**4. 特异性风险（Idiosyncratic Risk）**
- 个股特有波动
- 集中度风险（重仓股占比）

**输出示例：**
```
## 波动因子分析

### 风险暴露
| 因子 | 暴露度 | 风险等级 |
|------|--------|----------|
| 市场 Beta | 0.85 | 中 |
| 市值因子 | 0.60 | 中高（小盘倾向） |
| 价值因子 | -0.30 | 中（偏成长） |
| 波动率因子 | 0.45 | 中高 |

### 主要风险来源
1. **行业集中风险**：电子行业占比 35%，受科技股波动影响大
2. **集中度风险**：前十大持仓占比 55%，个股影响显著
3. **风格风险**：偏成长风格，在市场下跌时回撤可能较大

### 建议
- 如担忧波动，可考虑增加价值风格配置
- 关注行业分散度，避免过度集中
```

### 金融知识查询

AI 可利用自身知识库回答以下问题：

- 金融术语解释（如阿尔法、贝塔、夏普比率等）
- 基金类型说明（ETF、LOF、FOF、QDII 等）
- 投资策略解释（定投、轮动、对冲等）
- 市场机制说明（涨跌停、T+1、熔断等）

**当遇到不确定的最新信息时**，AI 应使用 WebSearch 或 WebFetch 工具进行查询验证。

### 深度调研

对于复杂问题，AI 可使用以下工具进行深度调研：

**1. WebSearch（网络搜索）**
- 搜索最新市场资讯
- 查询特定金融概念
- 获取行业研究报告

**2. WebFetch（网页抓取）**
- 获取特定网页内容
- 提取研究报告要点
- 获取基金公告信息

**3. Agent（子代理）**
- 对于需要多步骤研究的任务，可使用 Agent 工具进行并行探索

**调研示例场景：**
- "这只基金重仓新能源，最近新能源板块有什么政策？"
- "查找关于基金波动率因子的最新研究"
- "搜索 GitHub 上有哪些量化基金开源项目"

## 服务启动

### 检查服务状态

```bash
# 检查后端是否运行
curl -s http://localhost:8000/health
```

返回 `{"status": "healthy"}` 表示服务正常。

### 启动后端服务

如果服务未运行，Agent 应自行启动：

```bash
# 1. 进入项目根目录
cd <PROJECT_ROOT>

# 2. 安装依赖（如果需要）
pip install -r requirements.txt

# 3. 启动后端服务
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```

**注意：**
- 后端启动后，API 文档在 http://localhost:8000/docs

## 估值类型说明

系统根据基金类型自动选择最优估值方法：

| 估值类型 | 置信度 | 适用场景 |
|----------|--------|----------|
| `real_time_price` | 100% | 场内 ETF、LOF（直接取交易价格） |
| `index_based` | 85% | 指数基金、ETF 联接（基于跟踪指数） |
| `holdings_based` | 60-80% | 主动股票/混合型（基于持仓股票） |
| `benchmark_only` | 30% | 数据缺失（仅参考业绩基准） |

## 分析流程建议

### 完整基金分析流程

1. **基础数据获取**
   - 获取基金信息和持仓
   - 获取实时估值
   - 获取重仓股行情

2. **涨跌原因分析**
   - 计算持仓贡献度
   - 识别正/负贡献来源
   - 判断风格因素

3. **风险因子分析**
   - 识别行业暴露
   - 判断风格倾向
   - 评估集中度风险

4. **综合判断**
   - 结合市场环境分析
   - 提供投资建议

### 调研流程

对于需要外部信息的问题：

1. **明确问题**：确定需要查询的具体信息
2. **选择工具**：
   - 简单查询 → 直接使用 AI 知识
   - 最新资讯 → 使用 WebSearch
   - 深度内容 → 使用 WebFetch
3. **整合回答**：将调研结果与基础分析结合

## 注意事项

1. **服务依赖**: 所有功能依赖后端服务运行在 `http://localhost:8000`
2. **交易时间**: 估值数据仅在 A 股交易时段（9:30-15:00）有效
3. **QDII 时差**: 投资海外的 QDII 基金估值时间与 A 股不同步
4. **数据刷新**: 建议设置合理的刷新间隔（>=30 秒），避免被数据源限制
5. **免责声明**: 估值数据仅供参考，不构成投资建议

## 故障排除

**问题：API 调用返回空数据或错误**

```bash
# 1. 检查后端服务是否运行
curl http://localhost:8000/health

# 2. 检查基金代码是否正确
curl http://localhost:8000/api/funds/query/110022
```

**问题：估值数据为 null**
- 可能是非交易时间
- 基金可能是 QDII，需要等待海外市场开盘
- 数据源可能暂时不可用

**问题：后端启动失败**
```bash
# 检查依赖是否安装
pip install -r requirements.txt

# 检查端口是否被占用
lsof -i :8000
```
